  <action>
    1. Create scripts/launch_tradingview.sh:

       ```bash
       #!/usr/bin/env bash
       # Launch TradingView Desktop with Chrome DevTools Protocol debugging enabled.
       # Required by tradingview-mcp so Claude Code can inspect and modify the chart.
       #
       # Usage: ./scripts/launch_tradingview.sh
       # Port: 9222 (hardcoded — tradingview-mcp default)

       set -euo pipefail

       TV_APP="/Applications/TradingView.app/Contents/MacOS/TradingView"
       CDP_PORT=9222

       if [[ ! -x "$TV_APP" ]]; then
         echo "ERROR: TradingView Desktop not found at $TV_APP"
         echo "Download from https://www.tradingview.com/desktop/"
         exit 1
       fi

       echo "Launching TradingView Desktop with CDP port $CDP_PORT..."
       "$TV_APP" --remote-debugging-port=$CDP_PORT &

       echo "TradingView launched (PID: $!)"
       echo "CDP available at: http://localhost:$CDP_PORT"
       echo ""
       echo "Next: In Claude Code, run: tv_health_check"
       ```

       chmod +x scripts/launch_tradingview.sh

    2. Read ~/.claude/.mcp.json. If the file does not exist or does not contain a
       "tradingview-mcp" entry, add one. The entry format per tradingview-mcp docs:

       ```json
       "tradingview-mcp": {
         "command": "node",
         "args": ["/path/to/tradingview-mcp/build/index.js"],
         "env": {}
       }
       ```

       To find the installed path: check common locations:
       - ~/tradingview-mcp/build/index.js
       - ~/src/tradingview-mcp/build/index.js
       - ~/.npm/tradingview-mcp/build/index.js
       Run: `find ~ -name "tradingview-mcp" -maxdepth 5 -type d 2>/dev/null | head -3`
       to locate it. Use the found path. If not found, use placeholder
       "/PATH/TO/tradingview-mcp/build/index.js" and note it in mcp_setup_notes.md.

       When editing ~/.claude/.mcp.json: preserve ALL existing entries — only add
       the tradingview-mcp key if it is absent. Never overwrite existing keys.

    3. Create .claude/mcp_setup_notes.md documenting:
       - Whether tradingview-mcp was already present or newly added
       - The path used for the node args
       - Status: CONFIGURED / NEEDS_PATH_UPDATE (if placeholder used)
       - Instructions for operator if path update needed:
         "Clone https://github.com/tradesdontlie/tradingview-mcp, run npm install && npm run build,
          then update ~/.claude/.mcp.json tradingview-mcp.args[0] to the build/index.js path"

    NOTE: Do NOT restart Claude Code or shell — that is the operator's action.
    NOTE: mcp_setup_notes.md is NOT a README or documentation file in the prohibited
    sense — it is a machine-readable config audit log consumed by the checkpoint task.
