NT8 PARTIAL CLASS PATTERN (from 01-RESEARCH.md):
- Files in Indicators/ folder: NT8 auto-generates wrapper code for classes inheriting from Indicator.
  Creating partial class files in Indicators/ causes CS0101 "type already defined" errors.
- Correct pattern: AddOns/ folder avoids NT8 auto-generation. All partial class extensions go there.
- Indicators/DEEP6.cs must have: `public partial class DEEP6 : Indicator` (keeps `: Indicator` — this is the primary declaration)
- Every AddOns/DEEP6.*.cs must have: `public partial class DEEP6` (NO `: Indicator` — partial extensions never repeat the base class)
- All AddOns files must be in namespace `NinjaTrader.NinjaScript.Indicators`
- All AddOns files must have the same using declarations as DEEP6.cs (copy verbatim)

FIELD OWNERSHIP RULES (per D-03, D-08):
- Each field lives in exactly ONE partial class file — the file whose Run method uses it primarily
- Fields used across engines (e.g., _emaVol used by E1 and E7): assign to E1 (primary owner), E7 reads it cross-file (legal in partial classes — same instance)
- Session state fields (_vwap, _vsd, _vah, _val, _ibH, _ibL, _ibTyp, _ibDone, _ibConf, _dPoc, _pPoc, _pocMB, _pocMU, _dayTyp, _oPx, _sOpen, _iHi, _iLo, _cvd, _ibEnd): assign to E6 (they are session context, primarily consumed by E6 VP+CTX)
- Scoring state (_total, _sigDir, _sigTyp, _sigLbl, _lastSig, _feed): assign to Scorer
- SharpDX resources (_dxG, _dxR, ... _dwF, _fC, _fS, _fL, _dxOk): assign to Render
- WPF widgets (_hBdr, _pBdr, _tabBdr, _panelRoot, etc.): assign to UI
- E1 state: _fpSc, _fpDir, _fpSt, _cvd (CVD is accumulated in E1), _emaVol, _emaRng, _stkTier, _stkBull, _dQ, _ema20
- E2 state: _bV, _aV, _bP, _aP (DOM arrays), _imb, _imbEma, _pUp, _trSc, _trDir, _trSt, _iLong, _iShort
