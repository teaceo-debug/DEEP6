Per D-08: The delta trajectory divergence gate currently only exists as a concept inside the fading_momentum detector (variant 5). It must be applied as a universal filter to ALL 6 exhaustion sub-types.

The gate logic: exhaustion only fires when cumulative delta is fading relative to price direction. Specifically:
- If price direction is bullish (close > open), delta should be negative or decreasing (sellers losing conviction despite price rising = exhaustion of buyers who pushed price up)
- If price direction is bearish (close < open), delta should be positive or increasing (buyers losing conviction despite price falling = exhaustion of sellers who pushed price down)

Implementation:
1. Add a private function `_delta_trajectory_gate(bar: FootprintBar, config: ExhaustionConfig) -> bool` that returns True if the gate PASSES (signal allowed to fire).
2. Gate logic:
   - If `config.delta_gate_enabled` is False, always return True (gate disabled).
   - Compute `delta_ratio = abs(bar.bar_delta) / bar.total_vol` if total_vol > 0, else 0.
   - If `delta_ratio < config.delta_gate_min_ratio` (0.10), return True — delta too small to be meaningful, don't block.
   - If bar is bullish (close > open): gate passes if `bar.bar_delta < 0` (negative delta on green bar = fading buyers).
   - If bar is bearish (close < open): gate passes if `bar.bar_delta > 0` (positive delta on red bar = fading sellers).
   - If bar is doji (close == open): return True — no directional context.
3. Apply the gate at the TOP of `detect_exhaustion()`, right after the early returns for empty bars. If `_delta_trajectory_gate()` returns False, return empty list immediately.

EXCEPTION: Zero print (EXH-01) should NOT be gated — zero prints are structural gaps that exist regardless of delta. Add the gate check AFTER zero print detection but BEFORE variants 2-6. Restructure: detect zero prints first, then check gate, then detect variants 2-6. If gate fails, return only zero print signals (if any).

Update the fading_momentum detector (variant 5) to remove its redundant internal delta check since the universal gate now handles this. Keep the `abs(bar.bar_delta) > bar.total_vol * 0.15` threshold in fading_momentum as that is a STRONGER requirement specific to that variant (detecting pronounced divergence, not just any fading).
  </action>
