## Spacing Scale

All values are multiples of 4. The 8-point base scale applies throughout.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline chip padding, footprint cell inner margin |
| sm | 8px | Compact row spacing, tag padding, signal badge padding |
| md | 16px | Default panel padding, card inner padding, form field spacing |
| lg | 24px | Section padding, panel header padding |
| xl | 32px | Layout column gap, major widget separation |
| 2xl | 48px | Reserved — not used in Phase 11 dense layout |
| 3xl | 64px | Not used in Phase 11 |

Exceptions:

| Exception | Value | Reason |
|-----------|-------|--------|
| Touch targets (replay controls) | 44px min height | Accessibility — WCAG 2.5.5 |
| Footprint cell row height | 20px | Density requirement — 30 rows visible in fixed chart height |
| Signal feed row height | 48px | Accommodates two lines of text + badge at comfortable scan density |

Two-font system: Inter for all narrative/UI text, JetBrains Mono for all numeric and tabular data. No mixing within a single text element.

| Role | Font | Size | Weight | Line Height | Usage |

| UI label emphasis | Inter | 13px | 600 | 1.5 | Section headers, column headers, badge text |

| Numeric / price | JetBrains Mono | 13px | 400 | 1.0 | Price levels, bid/ask volumes, delta values, T&S entries |

## Footprint Custom Series Plugin Contract

LW Charts v5.1 has no built-in footprint renderer. This must be built as a custom series. The reference implementation is TapeFlow (https://github.com/ianfigueroa/TapeFlow) — study its Canvas overlay approach and ring buffer pattern before implementing.

### Data Shape

```typescript
// Per price level within a bar
interface FootprintLevel {
  price: number;       // exact price (NQ tick = 0.25)
  bidVol: number;      // contracts traded at bid
  askVol: number;      // contracts traded at ask
  delta: number;       // askVol - bidVol
  isImbalance: boolean;
  imbalanceRatio?: number;  // e.g. 3.5 = 350%
}

// Per bar — the unit passed to LW Charts custom series
interface FootprintBar {
  time: number;        // Unix timestamp (bar open)
  open: number;
  high: number;
  low: number;
  close: number;
  volume: number;
  delta: number;       // net bar delta
  poc: number;         // price of peak volume level
  levels: FootprintLevel[];
  signalType?: 'TYPE_A' | 'TYPE_B' | 'TYPE_C' | null;
  zoneOverlaps: ZoneRef[];  // zones active during this bar
}
```

### Renderer Responsibilities

The custom series renderer draws per bar:

1. **OHLC candlestick body** — standard candle using open/close, colored `#22c55e` (bullish) or `#ef4444` (bearish).
2. **Footprint cell grid** — for each `FootprintLevel` in the bar's visible range:
   - Row height: 20px
   - Bid volume text: left-aligned, JetBrains Mono 11px, color `#ef4444`
   - Ask volume text: right-aligned, JetBrains Mono 11px, color `#22c55e`
   - Background of cell: proportional fill — bid volume fills from left in `rgba(239,68,68,0.15)`, ask volume fills from right in `rgba(34,197,94,0.15)`
   - Imbalance highlight: when `isImbalance === true`, full cell background `rgba(163,230,53,0.25)` (lime wash)
   - POC row: full-width horizontal line `1px solid #facc15` at the poc price
3. **Delta footer** — below the candle body, delta value in JetBrains Mono 11px, `#22c55e` if positive, `#ef4444` if negative.
4. **Signal marker** — if `signalType` is set, render a colored triangle marker above the high:
   - TYPE_A: `#a3e635` (lime) upward or downward triangle, 8px
   - TYPE_B: `#facc15` (yellow)
   - TYPE_C: `#38bdf8` (sky)

### Price Axis Auto-Center Behavior

- On every new bar, if operator has not manually panned: scroll the chart to keep current price vertically centered in the 30-row visible range.
- Track manual pan with a `userHasPanned: boolean` state.
- Show a "Return to live" pill button (`#a3e635` text, `#16161f` bg, top-right of chart area) when `userHasPanned === true`. Clicking it resets pan and clears the flag.

### Zone Overlay Canvas Layer

The `ZoneOverlay` component renders on a separate `<canvas>` absolutely positioned over the LW Charts container. It does not use the LW Charts API for zone drawing — it draws direct 2D context bands.

| Zone Type | Fill | Border |
|-----------|------|--------|
| LVN | `rgba(56,189,248,0.08)` | `1px dashed rgba(56,189,248,0.4)` |
| HVN | `rgba(250,204,21,0.08)` | `1px dashed rgba(250,204,21,0.4)` |
| Absorption zone | `rgba(163,230,53,0.12)` | `1px solid rgba(163,230,53,0.5)` |
| GEX call wall | `rgba(34,197,94,0.08)` | `1px dashed rgba(34,197,94,0.5)` |
| GEX put wall | `rgba(239,68,68,0.08)` | `1px dashed rgba(239,68,68,0.5)` |

Zone bands span the full width of the chart at the zone's price range. The canvas must be kept in sync with LW Charts' coordinate system via `chart.priceToCoordinate()`.

### Row Anatomy

Each `SignalFeedRow` is 48px tall. Layout:

```

│ ▌  [TYPE_A]  ABSORBED @VAH  ·  21,450.25   08:32:14 │

  │      badge     narrative label    price      time
  │
```

Left border: 4px solid, color matches signal type. This is the primary visual differentiator.

Badge: shadcn `Badge`, 6px padding, 11px Inter 600, bg = type color at 20% opacity, text = type color full.

Narrative label: Inter 13px 400, `#e4e4e7`.

Price: JetBrains Mono 13px 600, `#fafafa`.

Time: JetBrains Mono 12px 400, `#6b7280`.

Score: JetBrains Mono 12px 400, color maps to score range — `>=80` uses `#a3e635`, `50-79` uses `#facc15`, `<50` uses `#6b7280`.

Engine agreement: Inter 12px 400, `#6b7280`. Lists contributing engine IDs.

### Animation: TYPE_A Pulse

When a new TYPE_A signal arrives:

1. The row enters from the top of the list (new entries prepend).
2. For exactly 1 second after arrival, the row's background pulses:
   - Keyframe: `0% { background: rgba(163,230,53,0) } 50% { background: rgba(163,230,53,0.18) } 100% { background: rgba(163,230,53,0) }`
   - Animation name: `typeAPulse`
   - Duration: 1000ms
   - Easing: `ease-in-out`
   - Iteration: 1 (not looping)
3. The 4px left border is always visible (not animated).
4. After 1 second the row returns to default surface background `#111118`.

TYPE_B and TYPE_C rows: no animation. Left border color + badge provides sufficient differentiation.

```css
@keyframes typeAPulse {
  0%   { background-color: rgba(163, 230, 53, 0); }
  50%  { background-color: rgba(163, 230, 53, 0.18); }
  100% { background-color: rgba(163, 230, 53, 0); }
}

.signal-type-a-pulse {
  animation: typeAPulse 1000ms ease-in-out 1 forwards;
}
```

Apply `signal-type-a-pulse` class to the row element via React `key` trick — assign a new key on arrival to force remount and restart animation.

### Scroll Behavior

- 20 rows visible at 48px row height = 960px scroll area height.
- New signals prepend to top of list.
- If user has scrolled down (viewing history), do NOT auto-scroll to top on new arrival. Instead show a "New signal" pill at top of the scroll area with count.
- Infinite scroll: load older signals in batches of 50 on scroll-to-bottom, fetching from FastAPI event history endpoint.

## WebSocket Protocol Contract

The UI consumes two FastAPI WebSocket endpoints (defined in Phase 9 EventStore):

| Endpoint | Data | Push frequency |
|----------|------|----------------|
| `ws://localhost:8000/ws/signals` | `SignalEvent` JSON | On every signal fire (TYPE_A/B/C) |
| `ws://localhost:8000/ws/bars` | `FootprintBar` JSON | On bar close + on DOM update (partial bars) |

The `useWebSocket` hook must:
- Reconnect with exponential backoff (1s, 2s, 4s, 8s, max 30s) on disconnect.
- Maintain a `status: 'connected' | 'reconnecting' | 'disconnected'` field exposed to the header strip.
- Not attempt reconnect if the browser tab is hidden (`document.visibilityState === 'hidden'`).

## Replay Mode Contract

Replay is triggered by selecting a historical session date (UI for date picker deferred — Phase 11 uses URL param `?session=YYYY-MM-DD`).

When in replay mode:
- WebSocket connection is replaced by an HTTP polling loop against `GET /api/replay/{session}/{bar_index}`.
- The replay controls strip becomes fully interactive.
- All real-time overlays (connection dot, live price) are replaced with replay context (bar timestamp, session date label).
- Signal feed shows only signals up to the current replay bar index.
- Zone overlays show only zones active at the current replay bar.

Playback auto-advance speeds:
- `1x` — 1 bar per second
- `2x` — 2 bars per second
- `5x` — 5 bars per second
- `auto` — advance on `requestAnimationFrame` as fast as rendering allows
