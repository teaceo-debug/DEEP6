<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→FastAPI | Signal/trade events arrive from the internal trading engine (same process) — not from the network in production, but the HTTP surface is exposed if uvicorn listens on 0.0.0.0 |
| weights→scorer | Weight file is loaded from disk and injected into scorer — tampered weight file = adversarial scoring |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | `deep6/api/routes/weights.py` POST /weights/deploy | mitigate | Require confirmation token (opaque UUID) in request body; token generated by system at training time and displayed to operator — prevents blind replays |
| T-09-02 | Information Disclosure | `deep6/api/store.py` signal_events table | accept | DB file on local disk only; no PII; trading signals are not secrets in single-user context |
| T-09-03 | Denial of Service | POST /events/signal called in hot loop at bar close | mitigate | EventStore insert is fire-and-forget wrapped in asyncio.shield — if DB is slow, signal pipeline is never blocked |
| T-09-04 | Elevation of Privilege | POST /weights/deploy without auth | mitigate | WFE gate enforced server-side; deploy endpoint returns 401 if DEPLOY_SECRET env var is set and header is missing |
</threat_model>
