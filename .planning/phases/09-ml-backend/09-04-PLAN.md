    `TierMetrics` dataclass:
    ```python
    @dataclass
    class TierMetrics:
        tier: str                # "TYPE_A", "TYPE_B", "TYPE_C"
        n_trades: int
        win_rate: float          # wins / n_trades
        profit_factor: float     # sum(wins) / abs(sum(losses)); inf if no losses
        sharpe: float            # mean(pnl) / std(pnl) * sqrt(252); 0 if std=0
        avg_pnl: float
        total_pnl: float
        regime: str | None       # None = all regimes; "ABSORPTION_FRIENDLY" etc for slices
        window: int              # 50, 200, or 500 (rolling window size used)

        def to_dict(self) -> dict: ...
    ```

    `PerformanceTracker` class:
    - `__init__(self, windows: list[int] = None)` — defaults to `[50, 200, 500]` (D-22)
    - `compute(self, trade_rows: list[dict]) -> list[TierMetrics]` — pure sync; no async needed:
      1. Filter to closed trades: `event_type in {"STOP_HIT", "TARGET_HIT", "TIMEOUT_EXIT", "MANUAL_EXIT"}`
      2. Sort by ts ascending
      3. For each tier in {"TYPE_A", "TYPE_B", "TYPE_C"}: for each window in self.windows:
         - Take last `window` closed trades for that tier
         - Compute TierMetrics: win_rate, profit_factor, sharpe, avg_pnl, total_pnl, regime=None
      4. For each regime in {"ABSORPTION_FRIENDLY", "TRENDING", "CHAOTIC"}:
         - Filter trades where `regime_label == regime` (column added when trade is recorded — see note below)
         - For each window: compute TierMetrics with `regime=regime`
      5. Return all TierMetrics as flat list

    Note on regime_label: trade_events table does NOT yet have a regime_label column. Add it:
    - Add `regime_label TEXT DEFAULT 'UNKNOWN'` to `TRADE_EVENTS_SCHEMA` in `deep6/api/store.py`
    - Update `insert_trade_event` to accept optional `regime_label` field
    - Update `TradeEventIn` schema: `regime_label: str = "UNKNOWN"`

    `async fetch_and_compute(self, store: EventStore, limit: int = 2000) -> list[TierMetrics]`:
    - Fetches from store, calls compute(), returns list

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| metrics endpoint→trade data | Performance metrics are derived from trade history — must not expose individual trade details in aggregated endpoint |
| E7 weight load→scorer | Weight quality score feeds into scoring — wrong weights silently degrade signal quality |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-13 | Information Disclosure | GET /metrics/signals returns P&L data | accept | Single-user local system; no PII |
| T-09-14 | Tampering | E7MLQualityEngine reads weights on every bar | mitigate | File read is O(file_size) — cache with 1-bar TTL; re-read only when file mtime changes. Prevents stale weights AND avoids repeated disk I/O |
| T-09-15 | Denial of Service | PerformanceTracker.compute on 2000 rows blocks | accept | Pure numpy computation on 2000 rows < 5ms; no async needed; acceptable in executor if necessary |
</threat_model>
