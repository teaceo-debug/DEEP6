    `DeployGate` class:
    - `__init__(self, wfe_threshold: float = 0.70, min_oos_trades: int = 200, weight_cap: float = 3.0)`
    - `evaluate(self, candidate: WeightFile, current: WeightFile | None, oos_counts: dict[str, int], confirmation_token: str, expected_token: str) -> DeployDecision`:
      1. If `confirmation_token != expected_token` → `DeployDecision(allowed=False, reason="Invalid confirmation token")`
      2. If `candidate.wfe is None or candidate.wfe < wfe_threshold` → `DeployDecision(allowed=False, reason=f"WFE {candidate.wfe} < {wfe_threshold}")`
      3. Per-signal OOS trade count check (D-17: 200+ OOS trades per signal before weight update):
         - For each signal in candidate.weights: if `oos_counts.get(signal_tier_mapping, 0) < min_oos_trades` → collect into `insufficient_signals`
         - `signal_tier_mapping`: map signal names to tiers that accumulate trades (e.g., any signal with TYPE_A/B history counts)
         - Simplified: check total OOS trades `sum(oos_counts.values()) >= min_oos_trades`; if not → `DeployDecision(allowed=False, reason=f"Insufficient OOS trades: {sum(oos_counts.values())} < {min_oos_trades}")`
      4. Weight cap check (D-15): for each (signal, weight) in candidate.weights where weight > weight_cap → collect into `violations`
         - If violations exist AND no override flag in candidate.metadata → `DeployDecision(allowed=False, reason=f"Weight cap exceeded: {violations}")`
      5. Build before/after: `{"before": current.to_json() if current else None, "after": candidate.to_json()}`
      6. All gates pass → `DeployDecision(allowed=True, reason="All gates passed", ...)`
    - `generate_token(self) -> str` — `secrets.token_hex(16)` — store last generated token in `self._pending_token`; endpoint shows this to operator

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| API→disk | Weight file written to local disk; must be atomic |
| operator→deploy endpoint | Operator provides confirmation token — must match server-generated token |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-09 | Tampering | `os.replace` write path | mitigate | Write to `.tmp` first, then `os.replace` — atomic on macOS/POSIX; no half-written state |
| T-09-10 | Spoofing | Sweep job ID | accept | Jobs are in-memory dict; job_id is UUID4; low risk in single-user context |
| T-09-11 | Repudiation | Weight deployment not audited | mitigate | Log deploy event with structlog: `log.info("weights.deployed", wfe=wfe, n_samples=n, by="operator")` |
| T-09-12 | Denial of Service | Concurrent sweep triggers | mitigate | Allow only 1 active sweep job at a time — return 409 if any job has `status=="running"` |
</threat_model>
