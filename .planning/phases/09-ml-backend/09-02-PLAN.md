Build LightGBM meta-learner and HMM regime detector — the two ML models at the core of Phase 9.

Purpose: The meta-learner optimizes per-signal weights by learning which signal combinations predict 3-bar returns. The HMM provides regime context that modulates those weights in real-time. Both must train in ThreadPoolExecutor to never block the event loop.

</objective>

    ```python
    # Normalize importance of the 8 category features to weights
    # Raw importance sum → normalize so mean = 1.0 (baseline = no adjustment)
    # Then cap at weight_cap (D-15: 3x baseline)
    category_features = ["cat_absorption", "cat_exhaustion", ..., "cat_poc"]
    importances = {f: model.feature_importances_[FEATURE_NAMES.index(f)] for f in category_features}
    total_imp = sum(importances.values()) or 1.0
    mean_imp = total_imp / len(importances)
    weights = {cat: min(imp / mean_imp, weight_cap) for cat, imp in importances.items()}
    # Map cat_absorption → absorption, cat_delta → delta, etc.
    signal_weights = {k.replace("cat_", ""): v for k, v in weights.items()}
    ```

    Create `deep6/ml/hmm_regime.py` using `hmmlearn.hmm.GaussianHMM`.

    `RegimeState` enum:
    ```python
    class RegimeState(str, Enum):
        ABSORPTION_FRIENDLY = "ABSORPTION_FRIENDLY"   # Low vol, balanced flow
        TRENDING = "TRENDING"                          # Directional delta, expanding range
        CHAOTIC = "CHAOTIC"                            # High vol, wide spread
    ```

    `HMMRegimeDetector` class:
    - `__init__(self, n_states: int = 3, covariance_type: str = "full", n_iter: int = 100, random_state: int = 42)`
    - Internal: `self._model: GaussianHMM | None = None`; `self._state_map: dict[int, RegimeState]` — maps HMM hidden state index → RegimeState
    - `_extract_hmm_features(signal_rows: list[dict]) -> np.ndarray` — builds (N, 5) float32 matrix with columns: `[atr_ratio, spread_proxy, trade_rate, delta_abs_mean, range_to_atr]`
      - `atr_ratio` = total_score / 100.0 (proxy until ATR available)
      - `spread_proxy` = 1.0 - engine_agreement (lower agreement = wider effective spread)
      - `trade_rate` = category_count / 8.0 (normalized)
      - `delta_abs_mean` = abs(direction) * engine_agreement
      - `range_to_atr` = total_score / 100.0 * category_count / 8.0 (composite)
    - `fit(self, signal_rows: list[dict]) -> None` — minimum 30 rows required; calls `GaussianHMM.fit(X)`; maps states to RegimeState by examining mean vectors:
      ```
      - Lowest spread_proxy → ABSORPTION_FRIENDLY
      - Highest delta_abs_mean → TRENDING
      - Highest spread_proxy → CHAOTIC
      ```
    - `predict_current(self, recent_rows: list[dict], n_recent: int = 20) -> RegimeState` — takes last n_recent rows, runs Viterbi decode via `model.predict(X)`, returns last predicted state. Returns `ABSORPTION_FRIENDLY` if model not fitted.
    - `async retrain(self, store: EventStore, loop: asyncio.AbstractEventLoop | None = None) -> None` — fetches last 30-day window (limit=5000) from store, calls `fit()` in `loop.run_in_executor(None, self.fit, rows)` — D-12: never blocks event loop; D-07 pattern
    - `is_fitted(self) -> bool` — `self._model is not None`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| EventStore→LGBMTrainer | Signal history read from SQLite — corrupted rows would produce wrong labels |
| model.pkl→scorer | Pickle deserialization of model file — malicious pickle = RCE |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-05 | Tampering | `deep6_model.pkl` pickle file | mitigate | Validate model file checksum (SHA256) stored in meta JSON; verify on load before using model |
| T-09-06 | Information Disclosure | LightGBM feature importance → weight file | accept | Weight file contains signal names but not trade P&L details; low sensitivity |
| T-09-07 | Denial of Service | LGBMTrainer training blocks if run_in_executor not used | mitigate | `_train_sync` is synchronous; it is called exclusively via `run_in_executor` — executor is NOT None (defaults to ThreadPoolExecutor); document this constraint in docstring |
| T-09-08 | Repudiation | HMM regime state changes not logged | mitigate | Log regime transitions with structlog: `log.info("hmm.regime_change", from_state=prev, to_state=new, ts=ts)` |
</threat_model>
