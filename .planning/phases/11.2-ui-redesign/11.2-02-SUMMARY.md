## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Motion `as const` readonly tuples rejected by TargetAndTransition**
- **Found during:** Task 1 typecheck
- **Issue:** `typeAFlashKeyframes` and `radialBloomKeyframes` used `as const`, making their array values `readonly` tuples. Motion's `TargetAndTransition` type requires mutable arrays.
- **Fix:** Replaced `as const` with explicit interface type annotations (`{ scale: number[]; filter: string[] }`) on the exported keyframe objects.
- **Files modified:** `dashboard/lib/animations.ts`
- **Commit:** 5a13973

**2. [Rule 2 - Missing data] `signalFlags` bitmask not in store â€” arc firing derived from `categoriesFiring[]`**
- **Found during:** Task 1 store inspection
- **Issue:** The plan specified reading a `signalFlags` bitmask from the store to fire individual arcs. The store's `ScoreSlice` exposes only `categoriesFiring: string[]` (category names) and `categoryScores: Record<string, number>`, not a raw bit field.
- **Fix:** All 44 arcs in a category ignite together when that category appears in `categoriesFiring`. The `SIGNAL_BIT_CATEGORIES` array remains correct for the future when a bitmask is exposed. `categoryScores` drives the middle ring opacity per-category as intended.
- **Files modified:** `dashboard/components/score/ConfluencePulse.tsx`
- **Commit:** 5a13973

**3. [Rule 2 - Missing data] `kronosConfidence` not in store â€” derived as `|kronosBias|`**
- **Found during:** Task 2, KronosBar store read
- **Issue:** `ScoreSlice` has `kronosBias: number` and `kronosDirection: string` but no separate `kronosConfidence` field.
- **Fix:** Confidence bar filled with `Math.abs(kronosBias)` as a 0â€“100 value. TODO comment added for future backend wiring.
- **Files modified:** `dashboard/components/score/KronosBar.tsx`
- **Commit:** 9876359

**4. [Rule 2 - Missing data] Zone registry not in store â€” ZoneList shows POC from latest bar only**
- **Found during:** Task 2, ZoneList store read
- **Issue:** Store has no `zones` array. `ZoneRef` type uses `LVN|HVN|ABSORPTION|GEX_CALL|GEX_PUT` (no VAH/VAL/POC). Bar data has `poc_price`.
- **Fix:** ZoneList derives a single POC zone from `bars.latest.poc_price` with distance-to-current-price in ticks. Empty state renders `NO ZONES` correctly. TODO comment added for Phase 5+ backend wiring.
- **Files modified:** `dashboard/components/zones/ZoneList.tsx`
- **Commit:** 9876359
