  <name>Task 1: Rewrite FootprintRenderer.ts — volume bars + bloom + markers</name>
  <files>dashboard/lib/lw-charts/FootprintRenderer.ts</files>
  <action>
    Major rewrite per UI-SPEC §4.2. Preserve the existing public API used by `FootprintSeries.ts` and `FootprintChart.tsx` — inspect both before writing, to confirm the method signatures (e.g., `draw(ctx, bars, pricesAxis, timeScale, options)`, constructor args, any `updateOptions` method). Do NOT change any signature used by locked files.

    Internal rewrite:

    1. **DPR handling** — follow existing Phase 11.1 Retina fix. Set ctx scale to `devicePixelRatio` at render start; compute logical widths/heights; use `ctx.imageSmoothingEnabled = false` for crisp bar edges.

    2. **Row height**: 16px (was 20 in v1 — UI-SPEC §3 densification).

    3. **Per-row rendering** (for each price level in a bar):
       - Compute `maxVolume = max(bid, ask)` across all rows in this bar
       - `cellCenterX = barLeft + barWidth/2`
       - `halfWidth = barWidth/2 - 2px` (2px gutter from centerline)
       - Bid bar: rect from `(cellCenterX - halfWidth * (bidVol/maxVolume))` to `cellCenterX - 1px`, height `rowHeight - 1px`. Fill `--bid` at 90% opacity (use `#ff2e63e6` or `ctx.globalAlpha = 0.9`).
       - Ask bar: rect from `cellCenterX + 1px` to `cellCenterX + halfWidth * (askVol/maxVolume)`. Fill `--ask` at 90%.
       - Volume numbers: render `bidVol` right-aligned inside the bid bar, and `askVol` left-aligned inside the ask bar, `11px JetBrainsMono 400`, color `#ffffff`, ONLY IF the bar fill width ≥ 32px; otherwise render the number OUTSIDE the bar in `--text` color.
       - Tabular numerics: use `ctx.fontVariantCaps = "tabular-nums"` where supported, or pre-pad numbers with space equivalents.

    4. **Imbalance bloom** — each cell carries a per-side `bidImbalance`/`askImbalance` flag (read from the same FootprintBar data that existing renderer uses; executor confirms the field name). When `imbalanceRatio ≥ 3.0`:
       - Fill at 100% opacity (not 90%)
       - Apply a bloom by drawing the bar twice: first a lower-alpha expanded copy (1.5px padding, alpha 0.35) as a pseudo-glow, then the crisp bar on top. This is the canvas equivalent of the CSS drop-shadow bloom (canvas filter "drop-shadow(...)" works in most browsers — use it if supported, fall back to the two-draw approach).

    5. **Stacked imbalance run line** — scan consecutive rows for same-side imbalance flags. For any run of length ≥ 3, draw a vertical 2px-wide `--lime` line on the imbalance side, spanning the run's vertical extent, positioned 1px inside the bar edge. This is the "runner" visual.

    6. **POC line** — bar's POC price is provided in the bar data. Draw a 1px horizontal line at the POC row's vertical center, full bar width, color `--amber`, with a 3px canvas shadowBlur in the same amber at 60% opacity (real bloom, not box-shadow).

    7. **Signal markers** — if the bar has a signal (TYPE_A/B/C tier), draw a thin 2px vertical line in the tier color (lime/amber/cyan) extending from the top row of the bar to 8px above the bar top, terminating in a 6×6 filled square in the tier color. No labels — narrative is in the signal feed. If multiple tiers fired on the same bar, draw the highest (TYPE_A > B > C) only.

    8. **Background grid** — vertical separators between bars at `--rule` (1px). No horizontal price grid.

    9. **Crosshair** — when crosshair event fires from LW Charts, render a 1px `--text-dim` dashed horizontal + vertical line.

    10. **Empty state** — if `bars.length === 0`, render centered text "AWAITING NQ FOOTPRINT" at 13px 500 JetBrainsMono, color `--text-mute`, letter-spacing 0.16em (use `ctx.letterSpacing` where supported or manual glyph-by-glyph placement). NO spinner. This replaces any existing empty-state text.

    Keep existing performance characteristics (Phase 11.1 DPR fix, 30-row density default). Do NOT change API surface consumed by `FootprintSeries.ts` or `FootprintChart.tsx`.
