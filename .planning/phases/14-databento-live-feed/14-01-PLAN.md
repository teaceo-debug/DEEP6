# Phase 14 Plan 01: Databento Live Feed

## Intent

Add a production-ready ``DatabentoLiveFeed`` that parallels
``deep6/data/rithmic.py`` in callback shape so the downstream pipeline
(BarBuilder ‚Üí LiveSignalPipeline ‚Üí LiveBridge) is agnostic to the data
provider. Rithmic keeps order execution; Databento owns market data.

Eliminates data drift between Phase 13 backtest (Historical MBO) and live
(Live MBO) ‚Äî both run through the same ``DOMState`` reconstruction and the
same ``FootprintBar.add_trade`` accumulator.

## Deliverables

1. ``deep6/data/databento_live.py``
   - ``_OrderBookState`` ‚Äî per-side price-level aggregate book, driven by
     MBO add/modify/cancel/trade events (D-09..D-11).
   - ``DatabentoLiveFeed`` class:
     - ``databento.Live(key=...)`` connection, dataset=``GLBX.MDP3``,
       schema=``mbo``, symbol=``NQ.c.0`` continuous (D-01..D-04).
     - Background-thread record callback posts onto ``asyncio.Queue`` via
       ``call_soon_threadsafe`` (D-21).
     - ``_drain`` task routes each record to book mutation or trade
       accumulation.
     - ``_dom_batcher`` emits a DOMState snapshot every 10 ms when the
       book has changed (D-22).
     - Aggressor derivation: side=A ‚Üí BUY, side=B ‚Üí SELL (D-13).
     - BarBuilder.on_trade is the RTH gate + FreezeGuard gate (D-15..D-17).
     - Disconnect/reconnect callbacks flip ``FreezeGuard`` state and log
       gap duration (D-18..D-20).

2. ``deep6/data/factory.py``
   - ``create_feed(source, config)`` dispatch:
     - ``"databento"`` ‚Üí ``DatabentoLiveFeed`` using
       ``config.databento_live_key()`` (prefers ``DATABENTO_API_KEY_GLBX``,
       falls back to ``DATABENTO_API_KEY``).
     - ``"rithmic"`` ‚Üí ``connect_rithmic(config)`` coroutine.

3. ``deep6/config.py``
   - Add ``databento_api_key_glbx`` field + ``DATABENTO_API_KEY_GLBX`` env
     loading + ``databento_live_key()`` helper.

4. Tests
   - ``tests/test_databento_live.py`` ‚Äî unit coverage (book ops, aggressor
     mapping, RTH gate, FreezeGuard, DOM dirty flag, int64 price scaling).
   - ``tests/test_data_factory.py`` ‚Äî factory dispatch + key selection.
   - ``tests/integration/test_databento_live_replay.py`` ‚Äî real MBO records
     from the local DBN slice (D-24), validates book reconstruction, trade
     accumulation, and DOM-dirty propagation end-to-end without a live API
     call.

## Constraints

- Rithmic path must remain byte-identical for ``--data-source=rithmic``.
- D-03 (aggressor gate), D-06 (RTH gate), D-17 (FreezeGuard) preserved on
  both paths.
- No blocking work in the databento record callback ‚Äî it runs on the SDK
  background thread. All DOM / bar work runs on the main asyncio loop.

## Out-of-scope

- Multi-instrument (ES, YM, RTY) ‚Äî v1 is NQ only.
- Custom snapshot intervals ‚Äî use Databento's native snapshot / initial
  book state.
- Moving ``_OrderBookState`` into a shared module with
  ``MBOAdapter`` ‚Äî both implementations are intentionally kept distinct
  (MBOAdapter uses bmoscon ``OrderBook`` for backtest-grade equivalence
  testing; live uses a plain-dict aggregate for lower per-event overhead).
