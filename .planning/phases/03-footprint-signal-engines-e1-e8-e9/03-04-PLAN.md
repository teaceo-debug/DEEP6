  <action>
Create `scripts/signal_correlation.py` with this structure:

1. **Reuse `build_bars()` from backtest_signals.py** (import it or duplicate the function — import preferred)
2. **Build a binary signal matrix**: For each bar, create a row of 0/1 values for each signal type:
   - All ImbalanceType variants (11 columns): 1 if that type appears in bar's imbalance signals
   - All DeltaType variants (13 columns): 1 if that type appears in bar's delta signals
   - All AuctionType variants (6 columns): 1 if that type appears in bar's auction signals
   - Total: ~30 signal columns
3. **Compute Pearson correlation** using `numpy.corrcoef()` on the binary matrix (transpose so signals are rows)
4. **Output CSV**: Write correlation_matrix.csv with signal names as header/index
5. **Print human-readable summary**:
   - Header: "SIGNAL CORRELATION ANALYSIS — {N} bars, {date range}"
   - Table of ALL pairs with |r| > 0.7, sorted by |r| descending
   - Table of ALL pairs with |r| > 0.5 (weaker correlations worth noting)
   - Count of signals that fire < 1% of bars (too rare for reliable correlation)

CLI args: `--start`, `--end`, `--output` (default: correlation_matrix.csv), `--bar-seconds` (default: 60)

Also support `--from-csv` mode that reads an existing backtest.csv instead of fetching from Databento (cheaper for re-analysis).

The script should work WITHOUT Databento if `--from-csv` is provided — this is important for testing without API costs.
