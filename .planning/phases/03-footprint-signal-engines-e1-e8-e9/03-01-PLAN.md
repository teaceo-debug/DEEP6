@dataclass(frozen=True)
class DeltaConfig:
    """Configuration for DeltaEngine — all tunable thresholds."""
    lookback: int = 20                    # Rolling window size
    tail_threshold: float = 0.95          # DELT-02: delta ratio for tail signal

    slingshot_quiet_ratio: float = 0.1    # DELT-08: max delta/vol for compressed bars
    slingshot_explosive_ratio: float = 0.4  # DELT-08: min delta/vol for explosion
    slingshot_quiet_bars: int = 2         # DELT-08: min quiet bars before explosion
    cvd_divergence_min_bars: int = 10     # DELT-10: min bars for CVD regression
    cvd_slope_divergence_factor: float = 0.3  # DELT-10: slope divergence threshold

**In imbalance.py**, change `detect_imbalances` signature:
- Replace kwargs with `config: ImbalanceConfig = ImbalanceConfig()` parameter
- Replace all references to `ratio_threshold`, `oversized_threshold`, `stk_t1/t2/t3` with `config.xxx`
- Add **IMB-02 Multiple** detection: after single imbalance scan, group buy_imb_ticks by tick value, any tick with count >= config.multiple_min_count fires MULTIPLE signal
- Add **IMB-07 Consecutive** detection: if prior_bar is provided, detect_imbalances on prior_bar, find ticks that are imbalanced in BOTH bars at same level. Fire CONSECUTIVE signal for each match.
- Add **IMB-09 Reversal** detection: if prior_bar is provided, check if prior bar had dominant buy imbalances and current bar has dominant sell imbalances (or vice versa). Fire REVERSAL signal.
- IMPORTANT: keep the existing `prior_bar` parameter (already in signature but unused for consecutive/reversal).

**In delta.py**, refactor DeltaEngine:
- Change `__init__` to accept `config: DeltaConfig = DeltaConfig()`; store as `self.config`
- Replace all hardcoded threshold values (0.95, 0.3, 0.1, 0.4, 10, 0.3) with `self.config.xxx`
- Add **DELT-03 Reversal** approximation: track max positive and max negative delta within process() by examining if delta sign differs from bar direction (close > open but delta < 0 = bearish reversal signal). This is the bar-level approximation since we don't have tick-level intrabar tracking.
- Add **DELT-07 Sweep** detection: if bar has >= config.sweep_min_levels price levels AND volume increases through the bar (last half > first half * config.sweep_vol_increase_ratio), fire SWEEP signal.
