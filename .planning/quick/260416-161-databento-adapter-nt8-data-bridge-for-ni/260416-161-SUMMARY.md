# Quick Task 260416-161: Databento Adapter + NT8 Data Bridge

**Completed:** 2026-04-16
**Commit:** acba4fb

## What Was Built

Two data adapters that feed real market data into the NinjaScript simulator, plus a universal NDJSON session loader that ties everything together.

### 1. Databento Adapter (`simulator/Adapters/DatabentoAdapter.cs`)

Converts Databento data files into the simulator's format:

- **OHLCV-1m CSV** → `SessionData` (direct load, synthesizes ticks from OHLC)
- **MBO trades CSV** → NDJSON (tick-level, with aggressor classification from side field)
- Handles Databento fixed-point pricing (1e9 scaling)
- Auto-detects column headers across schema variants

```bash
dotnet run --project ninjatrader/simulator -- databento NQ_ohlcv_1m.csv
dotnet run --project ninjatrader/simulator -- databento-mbo NQ_mbo_trades.csv
```

### 2. NT8 Data Bridge

**Server side** (`Custom/AddOns/DEEP6/Bridge/DataBridgeServer.cs`):
- TCP server that runs inside NT8 on port 9200
- Thread-safe: NT8 data thread writes, server thread sends
- Auto-accepts multiple simulator clients
- Sends session_reset on new client connect

**Indicator** (`Custom/Indicators/DEEP6/DataBridgeIndicator.cs`):
- Invisible indicator — add to any NQ chart to start bridging
- Forwards every OnMarketData tick (with BBO-based aggressor classification)
- Forwards every OnMarketDepth update (top 10 levels)
- Forwards bar close events from OnBarUpdate
- Configurable port (default 9200)

**Client side** (`simulator/Adapters/BridgeClient.cs`):
- TCP client that connects to the NT8 bridge
- Three modes: record-only, live-only, record+live
- Event-driven: OnTrade, OnDepth, OnBar, OnSessionReset
- Stats tracking (trades/depth/bars received)

```bash
# Live forward-testing (connect to NT8 on Windows):
dotnet run --project ninjatrader/simulator -- bridge 192.168.1.100:9200

# Record a session for later replay:
dotnet run --project ninjatrader/simulator -- bridge --record today-session.ndjson

# Replay the recorded session:
dotnet run --project ninjatrader/simulator -- replay today-session.ndjson
```

### 3. NDJSON Session Loader (`simulator/Adapters/NdjsonSessionLoader.cs`)

Universal loader for the common NDJSON format shared by all data sources:

- Parses depth, trade, bar, and session_reset events
- Attaches tick data to bars from interleaved trade events
- Synthesizes ticks from OHLC when no tick data available
- Multi-file loading for batch replay
- Detects bar period from timestamp spacing

```bash
# Replay captured NT8 sessions:
dotnet run --project ninjatrader/simulator -- replay --dir ninjatrader/tests/fixtures/sessions/

# Replay multiple specific files:
dotnet run --project ninjatrader/simulator -- replay session-01.ndjson session-02.ndjson
```

## Data Flow Architecture

```
Databento CSV ─────→ DatabentoAdapter ──┐
                                        │
NT8 DataBridge ────→ BridgeClient ──────┤── NDJSON ──→ NdjsonSessionLoader
                                        │                     │
CaptureHarness ────→ .ndjson files ─────┘                     ▼
                                                     NinjaScriptRunner
                                                           │
                                                     DEEP6Strategy
                                                    (full lifecycle)
                                                           │
                                                     Print log output:
                                                     signals, scores,
                                                     entries, vetoes
```
